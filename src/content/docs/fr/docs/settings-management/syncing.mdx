La synchronisation de vos paramètres signifie que si vous modifiez quelque chose dans Bonjourr sur une instance d'un navigateur, la modification sera automatiquement répercutée sur une autre instance.

<Aside type="caution">
	Si elle est correctement configurée, la synchronisation synchronise tous les paramètres de Bonjourr, à une exception notable près : les fichiers locaux importés comme arrière-plans ou icônes de liens. Il s'agit d'une limitation technique et d'une mesure de sécurité mise en place par les navigateurs concernant les extensions.
</Aside>

## Synchronisation dans un même navigateur {#same-browser-syncing}

Par défaut, si vous avez installé Bonjourr comme extension, vous verrez que l'option de type de synchronisation correspond à votre navigateur : **[Chrome Sync](https://support.google.com/chrome/answer/185277)**, **[Firefox Sync](https://www.firefox.com/features/sync/)** ou **[Edge Sync](https://support.microsoft.com/edge/sign-in-to-sync-microsoft-edge-across-devices)**.

Cela signifie que si :

- vous avez Bonjourr installé dans le même navigateur sur deux ordinateurs (par exemple, Firefox sur votre ordinateur portable + Firefox sur votre ordinateur de bureau),
- vous êtes connecté avec le même compte navigateur sur les deux navigateurs,

alors les deux instances de Bonjourr auront leurs paramètres synchronisés.

## Synchronisation entre différents navigateurs {#cross-browser-syncing}

Bonjourr propose deux façons de partager ses paramètres entre plusieurs navigateurs.

### GitHub Gist {#github-gist}

Bonjourr permet également de synchroniser ses données entre plusieurs navigateurs grâce à GitHub Gist (par exemple, si vous utilisez Chrome et Firefox en même temps).

<SmallDocsImage>
  ![Un exemple configuré de synchronisation GitHub Gist](@assets/documentation/settings-management/github-sync.png)
</SmallDocsImage>

Voici un guide étape par étape pour le configurer :

<Steps>

1.  #### Sélectionnez <kbd>GitHub Gist</kbd> dans « Type de synchronisation »

2.  #### Créez un token Gist

    Connectez-vous à [GitHub](https://github.com/) et créez un token d'authentification en suivant [ce guide sur docs.github.com](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens#creating-a-fine-grained-personal-access-token), ou en suivant les étapes montrées ci-dessous :

    <LazyVideo
        src="/documentation/settings-management/gist-sync.mp4"
        autoplay={false}
    />

3.  #### Collez votre token dans Bonjourr

    Retournez dans Bonjourr et collez votre token dans le champ <kbd>Jeton d'authentification</kbd>. Collez ce token dans autant d'instances de Bonjourr que nécessaire pour les synchroniser.

4.  #### Synchronisez !
	À partir de là, vous pouvez utiliser le bouton <kbd>Recevoir</kbd> pour télécharger vos paramètres depuis le serveur, ou le bouton <kbd>Envoyer</kbd> pour les envoyer. La zone <kbd>État du serveur</kbd> vous indique quand le dernier envoi a été effectué.
</Steps>

Bonjourr n'est pas affilié à GitHub, et aucune connexion vers `api.github.com` n'est effectuée tant que vous n'avez pas sélectionné cette option ET renseigné un token valide. Plus de détails sur notre [page de confidentialité](/fr/documentation/reference/privacy-policy/).

### URL distante {#remote-url}

Cette option est la plus manuelle de toutes, car elle nécessite une intervention pour que la synchronisation ait lieu.

Elle fonctionne essentiellement comme l'importation d'un fichier de paramètres depuis votre ordinateur, à la différence que vous le faites depuis une URL distante. Comme elle évite de devoir transférer le fichier de paramètres sur chaque appareil, cette méthode peut être pratique pour les utilisateurs avancés qui doivent configurer plusieurs instances de Bonjourr en même temps.

Hébergez simplement votre fichier de paramètres quelque part ([pastebin.com](https://pastebin.com/) peut être un bon choix), entrez son URL directe dans Bonjourr et cliquez sur le bouton <kbd>Recevoir</kbd>.
