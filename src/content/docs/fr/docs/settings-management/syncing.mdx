Syncing your settings means that if you change something in Bonjourr on one instance of a browser, it'll be replicated automatically in another browser.

<Aside type="caution">
	If set up correctly, syncing synchronises all of Bonjourr's settings, with one notable
	exception: custom backgrounds uploaded as local files.
</Aside>

## Same-browser syncing

By default if you've installed Bonjourr as an extension, you'll see that your synchronisation type option matches your browser: **[Chrome Sync](https://support.google.com/chrome/answer/185277)**, **[Firefox Sync](https://www.firefox.com/features/sync/)** or **[Edge Sync](https://support.microsoft.com/edge/sign-in-to-sync-microsoft-edge-across-devices)**.

This means that if:

- you have Bonjourr installed in the same browser on two computers (for example, Firefox on your laptop + Firefox on your desktop),
- you're logged in with the same browser account on both browsers,

Then both instances of Bonjourr will have their settings synchronised.

## Cross-browser syncing

Bonjourr has two ways to share its settings between multiple browsers.

### Github Gist

Bonjourr also has a way to synchronise its data accross multiple browsers thanks to GitHub Gist (for example, if you use Chrome and Firefox at the same time).

Here's a step by step guide on how to configure it:

<Steps>

1.  #### Select <kbd>GitHub Gist</kbd> in "Synchronization type"

2.  #### Create a Gist token

    Log in to [GitHub](https://github.com/) and create an authentification token by following [this guide on docs.github.com](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens#creating-a-fine-grained-personal-access-token), or by following the steps shown below:

    <video controls loop muted playsinline>
    	<source src="/documentation/overview/gist-sync.webm" type="video/mp4" />
    	Your browser does not support the video tag.
    </video>

3.  #### Paste your token in Bonjourr

    Go back to Bonjourr and paste your token in the <kbd>Authentification token</kbd> input. Paste that token in as many Bonjourr instances as you need to synchronise.

4.  #### Synchronise!
            From there, you can either use the <kbd>Get</kbd> button to download your settings from the server, or the <kbd>Send</kbd> button to upload them. The <kbd>Server status</kbd> area helps you by showing when the last upload was made.
</Steps>

Bonjourr is not affiliated with GitHub, and no connections to `api.github.com` are made until you select this option AND enter a valid token. More details on our [privacy page](/docs/reference/privacy-policy/).

### Remote URL

This option is the most manual of the bunch, as it requires intervention for the syncing to happen.

It's essentially the same as importing a settings file from your computer, except you will do it from a remote URL. As it removes the step of having to transfer the settings file to every single device, this could be handy for power users who might need to set up many instances of Bonjourr at once.

Simply host your settings file somewhere ([pastebin.com](https://pastebin.com/) can be a good choice), enter its direct URL in Bonjourr and click the <kbd>Get</kbd> button.
